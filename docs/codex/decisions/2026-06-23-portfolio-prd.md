# 个人作品集网站 PRD

日期：2026-06-23

## 1. 产品概述

本产品是一个 PC 优先的个人作品集网站 Demo，用于辅助王柯杰申请内容运营、新媒体运营及相关运营类实习岗位。

网站需要通过视觉作品、内容案例、实习经历和校园运营经历，证明用户不仅有简历经历，也具备可被查看、可被理解、可被验证的内容和运营产出。

## 2. 用户目标

求职方目标：

- 在众多运营实习求职者中建立差异化印象。
- 展示内容运营、新媒体运营、视觉表达、设计物料、活动社群和 AI 提效能力。
- 让招聘方快速理解用户的能力结构，并进一步查看作品和案例。

招聘方目标：

- 快速判断候选人是否适合内容/新媒体运营岗位。
- 查看真实作品、数据、经历和项目证据。
- 获取联系方式并跳转抖音主页。

## 3. 目标用户

主要用户：

- HR
- 运营岗位面试官
- 内容、新媒体、市场、品牌、产品运营团队负责人

使用场景：

- 简历筛选阶段，通过链接快速查看候选人亮点。
- 面试前，了解候选人的作品和经历。
- 面试中，作为案例展示材料使用。

## 4. 成功标准

首版 Demo 成功标准：

- 首屏能在 5 秒内建立专业、高级、视觉能力强的印象。
- 页面结构清晰，不像模板网站。
- 招聘方能快速看到用户的 6 类代表项目与经历。
- 摄影作品可点击查看完整大图。
- Works 和 Strengths 均支持详情弹层。
- 不展示手机号，只展示邮箱和抖音入口。
- PC 端视觉完整，内容不重叠、不溢出。

## 5. 范围

### 5.1 首版包含

- React + Vite 单页网站
- PC 端优先布局
- 暗色高级视觉系统
- 中英文并行标题
- Hero 全屏首页
- Profile/About 混合区块
- Selected Work 6 个主卡
- Strengths 能力卡
- Works 详情弹层
- Strengths 详情弹层
- Hero 摄影大图查看器
- Contact 整屏收尾页

### 5.2 首版不包含

- 完整移动端重设计
- 后端服务
- CMS 管理后台
- 在线表单提交
- 服务器部署
- 手机号展示
- 用户登录

## 6. 信息架构

页面导航：

1. `Profile / 关于我`
2. `Works / 精选项目`
3. `Strengths / 个人优势`
4. `Contact / 联系我`

页面区块：

1. Hero
2. Profile
3. Selected Work
4. Strengths
5. Contact

弹层类型：

- 图片大图查看器
- Works 项目详情弹层
- Strengths 能力详情弹层

## 7. 功能需求

### 7.1 Hero 首页

需求：

- 全屏高度展示。
- 使用城市夜景/建筑线条类摄影作品作为背景。
- 背景需加暗色遮罩。
- 顶部显示悬浮胶囊导航。
- 显示中英文姓名和求职定位。
- 显示一到两个行动按钮。
- 底部显示横向自动滚动摄影作品带。
- Hero 滚动作品只允许使用摄影作品。

交互：

- 点击滚动摄影作品，打开大图查看器。
- 大图查看器支持关闭、上一张、下一张。
- 大图中完整展示原始构图，不强行裁切。

验收：

- 首屏文字清晰可读。
- 背景图不喧宾夺主。
- 滚动作品不卡顿。
- 点击缩略图能正确打开对应大图。

### 7.2 Profile 关于我

需求：

- 融合 About 和 Profile。
- 展示头像或个人照片与摄影作品叠层。
- 展示个人简介、求职方向、所在地、可实习时间、邮箱。
- 展示关键成果数据。
- 展示经历时间线。

关键数据：

- 抖音单条最高播放 121w
- 单条最高获赞 8.1w
- 收藏 5000+
- 公众号推文 100+
- 商业拍摄 10+
- 校内触点 50+
- 校级人民奖学金 5 次

验收：

- 能清楚传达用户是内容与新媒体运营方向候选人。
- 学生身份、实习经历、校园经历不冲突。
- 不展示手机号。

### 7.3 Selected Work 精选项目与经历

需求：

展示 6 个主卡：

1. `Photography Works / 摄影作品`
2. `Douyin Content Case / 抖音内容案例`
3. `Design Materials / 设计物料`
4. `LSEG Internship / 产品与市场支持`
5. `Campus Media Operations / 融媒体内容运营`
6. `Campus Growth & Community / 校园增长与社群运营`

交互：

- 点击每张卡打开对应详情弹层。
- 详情弹层显示项目介绍、用户贡献、成果数据、相关素材。

验收：

- Works 不只是图片墙，也能展示无图运营经历。
- LSEG 和校园经历能通过信息可视化表达。
- 设计物料和内容运营不会被误写成同一项目。

### 7.4 Works 详情弹层

通用需求：

- 弹层在当前页面打开。
- 支持关闭。
- 内容区域可滚动。
- 保持暗色视觉系统。

摄影作品弹层：

- 展示精选摄影作品。
- 支持大图查看。
- 可显示作品标题、分类、简短说明。

抖音内容案例弹层：

- 展示抖音主页入口。
- 展示爆款数据。
- 展示代表视频截图或封面。
- 展示选题、拍摄、剪辑、发布、复盘逻辑。

设计物料弹层：

- 展示 Anker 展会相关素材。
- 展示物料清单和设计目标。
- 展示用户负责内容说明。

LSEG 弹层：

- 使用信息可视化表达，无需真实图片。
- 展示产品场景梳理、客户线索、资料整理、AI 提效。

融媒体弹层：

- 展示推文/H5/活动宣传截图。
- 展示内容统筹、发布流程和产出数量。

校园增长弹层：

- 展示美团校园大使经历。
- 展示触点网络、社群规模、活动 SOP。

### 7.5 Strengths 个人优势

需求：

展示 5 个能力卡：

1. 内容策划与传播
2. 摄影视觉表达
3. 设计物料与品牌呈现
4. 活动运营与社群触达
5. AI 与数据提效

交互：

- Strengths 也需要详情弹层。
- 至少内容策划、摄影视觉、设计物料 3 张卡必须可点击展开。
- 若实现成本可控，5 张均可点击。

详情弹层内容：

- 能力说明
- 证据数据
- 代表经历
- 关联 Works 项目

验收：

- 能力卡不是空泛技能罗列。
- 每个能力都有简历或作品证据支撑。

### 7.6 Contact 联系我

需求：

- 整屏收尾页。
- 展示邮箱。
- 展示抖音主页入口。
- 不展示手机号。
- 可选简历下载入口。

交互：

- 邮箱可复制或打开邮件客户端。
- 抖音入口跳转外部链接。

验收：

- 招聘方能清楚知道如何联系用户。
- 联系区不泄露手机号。

## 8. 内容需求

已知内容来源：

- 简历：`F:\大学\实习\内容运营方向.docx`
- 作品素材：`F:\作品材料`
- 摄影作品：`F:\作品材料\摄影作品`
- 商业拍摄：`F:\作品材料\商业拍摄`
- 设计物料：`F:\作品材料\设计物料（Anker展会）`

待补充：

- 头像或职业照
- 抖音主页链接
- 抖音案例截图
- H5 或公众号推文截图
- 美团校园宣传截图
- 公开展示邮箱确认
- 简历下载版本

## 9. 非功能需求

性能：

- 不直接加载超大原图。
- 首屏图片需使用压缩版本。
- 大图查看器可使用较高清版本，但仍需控制体积。

可访问性：

- 正文对比度满足可读性。
- 弹层可关闭。
- 交互元素有明显 hover/focus 状态。

维护性：

- 项目数据应尽量结构化，便于后续增删作品。
- 素材路径和内容文案分离。
- 后续可扩展移动端布局。

## 10. 技术约束

- 使用 React + Vite。
- PC 端优先。
- 版心约 `1700px`。
- 本地 Demo 优先，部署后置。
- 原始素材目录不得被修改。
- 实现阶段应在项目内生成网页用素材副本。

## 11. 验收清单

首版完成时需检查：

- Hero 背景正常显示。
- Hero 摄影滚动带只包含摄影作品。
- 图片点击后可完整查看大图。
- 6 个 Works 主卡都存在。
- Works 主卡都能打开详情弹层。
- Strengths 能力卡能打开详情弹层。
- Contact 包含邮箱和抖音入口。
- 页面不展示手机号。
- PC 端 1440px、1700px、1920px 宽度下布局正常。
- 不出现文字重叠或溢出。
- 暗色下正文可读。
